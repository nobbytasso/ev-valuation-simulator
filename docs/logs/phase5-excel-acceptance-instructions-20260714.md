## 2. Excel目視検収の手順(ユーザー実施分)

所要: 15分程度。Excel(またはNumbers/LibreOffice)がインストールされたこの Mac 上で実施する。

### 2.0 準備

1. **既存データの退避(重要)**: 検収はまっさらな状態から行うため localStorage をクリアする。**残したいシナリオがある場合は先に各シナリオを「エクスポート」でJSON保存しておく**(検収後にインポートで戻せる)。
2. `npm run dev` でアプリを起動 → `http://localhost:5173/#/`
3. ブラウザの DevTools コンソールで `localStorage.clear()` → リロード。ヘッダに「ダミーデータ」バッジが出ていることを確認。

### 2.1 検収用データの作成(完了報告の手順+セクター別ブロック確認用に1件追加)

1. シナリオ一覧で6セクターを順に選び「新規作成」(デフォルト値のまま): SaaS(日本)・創薬・医療機器・メディアテック・EC・D2C・クライメートテック。
2. **SaaS シナリオを1件「複製」する**(同一セクター2件のペアを作る。比較Excelのセクター別ブロック確認用)。
3. `/#/portfolio` で3銘柄を追加し、A社・B社は行のセレクトからシナリオへ紐付ける:
   - A社: SaaS、投資額 300、持分 5%、投資日 2024-01-01 → SaaSシナリオに紐付け
   - B社: 創薬、投資額 300、持分 5%、投資日 2023-06-01 → 創薬シナリオに紐付け
   - C社: 医療機器、投資額 300、持分 5%、**投資日未入力・紐付けなし**

### 2.2 ファイル① シナリオ単票(SaaSシナリオ詳細 →「Excelエクスポート」)

Excel で開いて確認:

- [ ] ファイル名 = `{シナリオ名}.xlsx`。シートが「結果」「前提条件」の2枚。文字化けなし・装飾なし(色/罫線なしのプレーン)
- [ ] 「結果」: EVレンジ = 悲観 **6,500** / ベース **10,400** / 楽観 **15,600**(百万円)。空きセルに `=SUM()` 等を打ち、**数値として計算できる**こと(文字列ではない=検収条件)
- [ ] 簡易DCF(auxiliary)**1132.4** 前後、Rule of 40 **35**(pt)
- [ ] VC法ブロック(入力+悲観/ベース/楽観の出力表、含意IRR **58.5%** 前後)
- [ ] 感度分析の上位10行(P5-8)と、資本政策のラウンド表・持分推移(デフォルトはラウンドなし=空またはその旨)が含まれる
- [ ] 「前提条件」: 全入力が日本語ラベル+単位付きで並び(%は百分率数値)、**ベンチマーク状態「ダミーデータ」の注記**がある

### 2.3 ファイル② 比較表(一覧で SaaS×2+創薬+医療機器 の4件にチェック →「選択したシナリオを比較」→「Excelエクスポート」)

- [ ] 5件目にチェックを入れようとすると**入れられない**(上限4件、P5-6)
- [ ] ファイル名 = `シナリオ比較_4件.xlsx`。「比較」「前提条件」の2シート
- [ ] 「比較」: 列=4シナリオ。EV3点(数値セル)・VC法(含意IRR **58.5** が全列同値=目標倍率と年数のみに依存する式の性質どおり)・期待IRR/MOIC は「—(自ファンドの出資がありません)」等の理由付き表示
- [ ] **「SaaS(日本)別指標」ブロックが出現**し(同一セクター2件のため)、keyMetrics・主要ドライバーが SaaS の2列にのみ値を持つ(創薬・医療機器の列は空欄)。創薬・医療機器のブロックは出ない(各1件のため)
- [ ] 「前提条件」: 「■ セクター: シナリオ名」見出しで4シナリオ分が縦に連結
- [ ] 画面の比較テーブルの数値と Excel の数値が一致する(抜き取りで数カ所)

### 2.4 ファイル③ ポートフォリオサマリ(`/#/portfolio` →「Excelエクスポート」)

- [ ] ファイル名 = `ポートフォリオサマリ.xlsx`。「サマリ」「前提条件」の2シート
- [ ] A社: 時価ベース **520**(= EVベース10,400 × 5%)、MOIC **1.73x** 前後、IRR は数値(投資日 2024-01-01 → 評価基準日=検収当日に依存するため、完了報告の 24.3% から日数分ずれるのは正常)
- [ ] B社(創薬): EVが負のため時価・MOICが**負値で表示され、隠されていない**こと。IRR「—(算出不能)」
- [ ] C社: 時価3点とも **300**(=投資額)、評価方法列「**コスト評価**」、IRR「—(投資日未設定)」
- [ ] ファンド合計行: 投資額合計 **900**、時価総額3点が銘柄の合計と一致、ファンドIRR「—(投資日未設定の銘柄があります)」、「コスト評価銘柄を含む」注記
- [ ] 「前提条件」: 評価基準日(検収当日)・銘柄別の紐付けシナリオと exitEvSource・コスト評価銘柄一覧(C社、理由=未紐付け)・ベンチマーク状態(ダミーデータ)
- [ ] 画面のポートフォリオ表示と Excel の数値が一致する
