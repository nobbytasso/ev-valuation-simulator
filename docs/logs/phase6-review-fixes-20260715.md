## 第2回指摘の対応

日付: 2026-07-15
指摘元: ユーザーによる実機デザインレビュー(第2回)
設計: `docs/phase6-spec.md` §12.2

### 指摘

画面が広いディスプレイ(ウィンドウ幅 約1650px)でコンテンツが全幅に伸びる。コンテンツ実効幅を約1450px程度(提示スクリーンショット相当)で固定してほしい。

### 対応

`src/ui/layout.css` のみを変更(§12.2 の設計どおり。テーマトークン・エンジン・store・golden には触れていない)。

- `:root`(テーマスコープ外)に共通カスタムプロパティ `--layout-max-width: 1440px` を新設。
- `.app-main` に `max-width: var(--layout-max-width); margin-inline: auto;` を追加(既存の `padding: 1.5rem` は維持)。
- `.app-header` はバーの背景・下罫線を全幅のまま、`padding: 0.75rem max(1.5rem, calc((100% - var(--layout-max-width)) / 2));` に変更し、内容(ブランド・ナビ・トグル群)を `.app-main` の内容と同じ左端に整列させた(DOM変更なし。§12.2で推奨されていた方式)。
- `Layout.tsx` の変更は不要だった(CSSのみで実装可能なため)。

コミット: `fee5b65`

### 検証結果

- `npm run typecheck`: Green
- `npm run lint`: Green(既存の pre-existing warning 4件のみ。本修正による新規warning/errorなし)
- `npm run test`: **475件 Green**(51ファイル。既存テストに影響なし)
- `npm run test:e2e`: **29本 Green**(E2E期待値の修正は不要だった。C2-1のテーマ切替レイアウトシフト検証も両テーマ同幅のため影響なし)
- 実ブラウザ確認(Playwright/Chromium、ビューポート幅1900px、シナリオ一覧・シナリオ詳細画面をダーク/ライト両テーマでスクリーンショット取得。スクラッチ領域に保存、リポジトリには非コミット):
  - `.app-main` のbounding boxを実測: `x=206, width=1488`(1440pxコンテンツ + 左右padding 1.5rem=24px×2の合計。box-sizingがcontent-boxのため)。左右マージンはともに206pxで完全対称(中央寄せ)。
  - `.app-header__brand`(ヘッダ内容の開始位置)は `x=230` で、`.app-main` の内側パディング後の実コンテンツ開始位置(206+24=230)と一致。ヘッダ内容とメインコンテンツの左端が正しく整列していることを数値で確認。
  - ダーク/ライト両テーマで同一の数値(x=206, width=1488, 右マージン206)を確認。テーマによる幅の差異なし。
  - スクリーンショットで、シナリオ一覧テーブルパネルが画面中央に幅約1440pxで収まり、左右に均等な余白があることを目視確認。
  - 確認後、dev サーバーは停止済み。

### 備考

- `.app-main` の実測幅(1488px)は §12.2 の設計値1440pxに padding(1.5rem×2=48px)が加算されたもの。設計は「既存の `padding: 1.5rem` は維持」「max-width: 1440px」を明示しており、box-sizingの変更は設計に含まれていないため、実装は設計を忠実に再現した結果。設計が許容する「±10%の範囲での微調整」の枠内(1488は1440の+3.3%)であり追加調整は行っていない。
- 新たな未確定事項の発生なし。設計からの逸脱なし。
