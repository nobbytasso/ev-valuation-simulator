## 実装内容の要点

### T1: 評価ファサードとラベル基盤(C2)
- `evaluateScenario`(`src/ui/scenarioEvaluation/`)がSectorViewやbuildTornadoRowsと同じswitchディスパッチパターンでセクター横断評価を提供。
- `KEY_METRICS_LABELS`・6セクターの`<sector>FieldLabels.ts`は各Form.tsxの実文言と一致させ、テストで入力全キーの網羅を担保。

### T2: シナリオ並列比較ビュー(C3)
- URLクエリ(`?ids=`)で選択状態を保持、読み取り専用。上限4件(P5-6)。
- 共通ブロック(EV3点・VC法・期待IRR/MOIC)は常時表示、セクター別ブロックは同一セクター2件以上のときのみ(P5-5)。創薬はパス形式driverIdのため品目数・合計ピーク売上・割引率(ベース)の3行固定。
- EVレンジのグループ棒チャート(Recharts)は数表と同じ`evaluateScenario`結果を共有。

### T3: 仮想ポートフォリオ集計(C4, C5)
- `PortfolioHolding` v1→v2マイグレーション(`investmentDate`追加、null補完・冪等・両経路テスト)。
- 未紐付け・評価不能・シナリオ削除済みはコスト評価(投資額で代替、P5-1)。時価はEV×持分(P5-2)。投資日未設定・当日投資はIRR「—」(P5-3)。評価基準日は今日(UI層、P5-4)。
- ファンド単位集計: 時価総額は3点、IRR/MOICはベース基準のみ(P5-9)。投資日未設定銘柄が1件でもあればファンドIRR算出せず。

### T4: Excelエクスポート(C6, C7)
- `src/ui/excel/`にxlsx importを隔離。3種のワークブック(シナリオ単票・比較表・ポートフォリオサマリ)を純粋関数として実装、XLSX.write→readの往復読み戻しテストで数値セル型・シート構成を検証(計47件)。
- シナリオ単票は感度上位10行+資本政策のラウンド表・持分推移を含む(P5-8)。
- 「値の素性」の明示: コスト評価バッジ、投資日未設定の「—」表記、ベンチマークデータ状態(dummy/production/unknown)を前提条件シートに明記。

## 実ブラウザでの検収用データ作成・検証(手順と結果)

C6/C7完了後、開発サーバー(`npm run dev` + Playwright)で以下を実施し、3種のExcelファイルを実際に生成・読み戻して内容を確認した(一時検証スクリプトはコミット対象外、スクラッチ領域に生成物のみ保存)。

### 検収用代表データの作成手順(再現可能)

1. `http://localhost:5173/#/` でシナリオ一覧を開き、localStorageをクリアした状態から開始する。
2. セクターセレクトで以下6セクターを順に選び「新規作成」をクリックする(デフォルト入力値のまま): SaaS(日本)・創薬・医療機器・メディアテック・EC・D2C・クライメートテック。
3. 1件を開き「Excelエクスポート」→ **①シナリオ単票**。
4. シナリオ一覧に戻り、6件中4件(上限)にチェックを入れ「選択したシナリオを比較」→比較ページで「Excelエクスポート」→ **②比較表**。
5. `/#/portfolio` を開き、3銘柄を追加する: A社(SaaS、投資日2024-01-01)・B社(創薬、投資日2023-06-01)・C社(医療機器、投資日未入力=未紐付けのまま)。A社・B社は行のセレクトから対応するシナリオへ紐付ける。「Excelエクスポート」→ **③ポートフォリオサマリ**。

### 生成ファイルの読み戻し結果(抜粋)

- **①シナリオ単票**(SaaSデフォルト): 「結果」42行・「前提条件」23行。EVレンジ `[6500, 10400, 15600]`(百万円、数値型)、簡易DCF `1132.4`、Rule of 40 `35pt`、VC法入力・出力・含意IRRまで一貫して出力を確認。
- **②比較表**(4セクター混在): 「比較」10行。EV・VC法含意IRR(58.5%、4列とも同値=exitEnterpriseValueに非依存という式の性質どおり)・期待IRR/MOICの「—」理由注記まで一致。同一セクター2件以上がないためセクター別ブロックは出力されない(P5-5の仕様どおり)。
- **③ポートフォリオサマリ**: A社(紐付け済み)は時価ベース `520百万円`(EVベース10400×持分5%)・MOIC `1.73x`・IRR `24.3%`。B社(創薬、紐付け済み)はEVが負(rNPVの性質上、デフォルト入力で負値になり得る。engine-spec §2.2の仕様どおり)のためMOICが負・IRRは「—(算出不能)」(正負両方のCFが存在しないため)。C社(未紐付け)はコスト評価(時価=投資額300百万円固定)・IRR「—(投資日未設定)」。ファンド合計行は投資額合計900、時価総額3点、投資日未設定銘柄(C社)が1件あるためファンドIRRは「—(投資日未設定の銘柄があります)」+コスト評価銘柄を含む旨の注記。
- 比較表のB社(創薬)EVベース `-1818.84` と、ポートフォリオのB社時価ベース `-90.94`(= `-1818.84 × 5%`)が一致し、EV×持分の計算が両画面間で整合していることを確認した。

**ユーザーによる目視検収(Excelで実際に開いての確認)は本報告の対象外**であり、`docs/phase5-spec.md` §9・§6の指示どおりユーザー側で実施する。上記手順・生成物の内容はその検収を代替するものではなく、実装が意図どおり動作していることの一次確認である。

## 未確定事項・実装判断メモ(金額計算に関わらないもの)

以下はいずれも表示・レイアウト上の実装判断であり、金額計算のロジックには影響しない。仕様(phase5-spec.md)に明記がなかったため、既存の規約(ラベルの一貫性、値の素性の明示)に沿って妥当な値で仮決めした。ユーザー確認が必要と判断した場合は別途申し出る。

1. **創薬 `commercialization.milestones[]` の前提条件シート展開**: フィールドラベル表(`SectorFieldLabelTable`)には正式なラベルエントリを設けず、Excel生成側で既存の`PHASE_LABELS`と「金額」表記を直接使って展開した(疎な二段ネスト配列のため、独立ラベル化するほどの複雑性がないという判断。`drugDiscoveryFieldLabels.ts`にコメントで明記済み)。
2. **個別銘柄IRRの「当日投資」時の理由文言**: `investmentDate`が評価基準日と一致(t=0)の場合、理由を「当日投資のため未定義」とした。spec §3.3は「IRRは『—』」とのみ記載し具体文言は未指定。
3. **比較表Excelのセクター別ブロックの列レイアウト**: 画面(ScenarioComparePage)ではセクター別ブロックはそのセクターの列のみを並べるサブテーブルだが、Excelでは全列(全シナリオ)を維持し非該当セクターの列は空セルとした(1シートで一貫した表形式を保つための実装選択。値は画面と完全一致)。
4. **比較表・ポートフォリオExcelのベンチマークデータ状態取得**: エクスポートボタンを押した時点で非同期フェッチが完了していない場合(通常は数ms内に完了)、該当セクターの状態は「不明(未取得)」と表示する(dummyでもproductionでもない安全側のフォールバック)。
