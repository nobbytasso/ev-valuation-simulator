保存先: `/private/tmp/claude-501/-Users-nmorii-Projects-ev-valuation-simulator/eab3ede8-111c-4af9-863c-c47ba7de3c02/scratchpad/phase6-review/`(セッション終了で揮発する一時領域。ユーザーが実機で最終確認する際はブラウザで直接操作すること)

**撮影時に踏んだ落とし穴(再発防止のため記録)**: `page.screenshot({ fullPage: true })` は撮影直前に内部的にビューポートをリサイズする。このページにはRecharts(`ResponsiveContainer`、内部で`ResizeObserver`を使用)のチャートが複数あり、このリサイズがResizeObserverのコールバックを再発火させ、チャートのバーがアニメーション未収束(高さ0近辺)の一瞬を撮影してしまうことがあった(比較ビューのEVレンジチャートで発生・再現)。**製品側のバグではない**(通常のビューポート内表示・要素個別スクリーンショットでは正しく描画される。`elementFromPoint`・`getComputedStyle`で実際の描画属性を確認し、CSS変数`var(--color-status-bad)`等も正しく解決されていることを確認済み)。対策として、撮影前に手動でビューポート高をコンテンツ全高までリサイズしてから`fullPage: false`で撮る方式に切り替えた。

## 3. 実装中の判断メモ(未確定事項・仕様外判断の報告)

`docs/phase6-spec.md` / 裁定表に明記のない箇所で、実装上やむを得ず行った判断を以下に記録する。金額計算ロジックに関わるものはない(すべてUI表示・演出上の判断)。

1. **EV≤0警告の新規実装**: `phase6-spec.md` §5.1は「EV≤0の警告(創薬・クライメート)|bad(**既存**警告の色格上げ)」としていたが、調査の結果このUI警告は実際には未実装だった(エンジン側のコメント「EVは負を許容(UIで警告)」のみで、UIに反映されていなかった)。既存warningが存在しない以上「色格上げ」は行いようがないため、`EvRangeResult.tsx`にbad色の新規警告(`企業価値(ベース)が0以下です。...`)を実装した。既存のP6-5裁定表にない新しい判定閾値(0以下)は発明していない(EV≤0は既に §5.1 の規則として明記済みの境界)。
2. **円形ゲージのstatus**: 明示規則があるもの(VC法成立不可→bad、EV≤0→bad)以外は`neutral`固定とした。Rule of 40ゲージ・期待IRR/MOICゲージに新しい閾値(good/caution/badの境界)を発明していない(P6-5裁定表はベンチマーク対比の25指標のみを対象としており、ゲージ自体の判定基準は定義されていないため)。
3. **ライトの`--color-accent-contrast`を白から濃色シェード(#3a2933)へ変更**: 淡いピンク/警告色の地に白文字はAA(3:1)を満たさなかったため、C11のコントラストテストをGreenに保つ範囲でトークン値を調整した(要件§5制約1の機械化)。パステルの人格自体は変更していない。
4. **日英キャプションの適用範囲**: `docs/phase6-spec.md` §6.3が列挙する7見出し(結果/VC法/資本政策/感度分析/ベンチマーク比較/比較/ポートフォリオ)に加え、後者2つはページ本体のh1見出しに適用した(ページ内に該当するh2が存在しないため)。
5. **「基準日(as_of)」の表示位置(D-13)**: `phase6-spec.md`の文言「文言を「取得日」→「基準日(as_of)/取得日」に整理」を、出典行(エントリ単位)に両方を併記する形で実装した(セクション単位で1箇所にまとめる案もあったが、引用元を読む際に両日付が同じ行にある方が実用的と判断)。
