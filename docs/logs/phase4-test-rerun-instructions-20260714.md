## B. 実ブラウザ確認の手順

1. dev サーバー起動(バックグラウンド): `npm run dev` → `http://localhost:5173/`(HTTP 200 を確認)
2. ブラウザ起動(用途で使い分け):
   - **対話確認(ユーザー向け)**: `npx playwright open --browser=chromium "http://localhost:5173/#/"` — ヘッド付き Chromium が開く。DevTools コンソールを開いたまま §C を操作し、エラー・警告が出ないことを確認する
   - **スクリーンショット(エージェントセッション向け)**: `npx playwright screenshot --browser=chromium --full-page "<URL>" <出力先>.png` — 出力先はセッションのスクラッチ領域(リポジトリ内に画像を置かない)
3. URL 形式: ルーティングは **HashRouter**(`src/App.tsx`)。シナリオ詳細は `http://localhost:5173/#/scenarios/<id>`。id は localStorage 依存のため、まず一覧(`/#/`)でセクターを選んで「新規作成」し、遷移先のURLを使う
4. コンソールエラーの機械的な収集が必要な場合(スクリーンショットでは拾えない)は §D の @playwright/test 化で `page.on('console')` を使う
5. 自動化スクリプトを書く場合は、実検証で判明した3つの落とし穴(トルネード表はspan降順で自動ソートされるため行はラベルで追跡する/HashRouterのハッシュ遷移はフルリロードを伴わないためlocalStorage直接書き換え後は明示的に `page.reload()` する/テーマトグルはaria-label「テーマ切替」で照会する)を必ず踏まえること。詳細: `docs/logs/phase4-browser-verification-20260714.md` §3

## C. 確認チェックリスト(完了報告 §3 で未検証だった観点)

### C-1. 感度分析トルネード(C4。全6セクターで表示、重点は創薬)

- [ ] 6セクター各1シナリオを新規作成し、詳細ページに感度分析セクションが表示され、トルネードチャート(Recharts横棒)が描画される。コンソールエラーなし
- [ ] バーが span 降順に並ぶ。基準線=ベースEV
- [ ] 創薬: 品目を追加して総ドライバー数を11件以上にし、上位10件+「残りn件を表示」トグルが機能する
- [ ] ドライバー毎の変動幅(%)を変更すると再計算される。創薬 `割引率(ベース)` の変動幅は「±2pt(固定)」表示で編集不可
- [ ] 縮退 span=0 の表示: 創薬で `上市年=2` にする(±20%が四捨五入で消える)→「この変動幅では感度なし」グループにラベル列挙される(非表示にならない)
- [ ] SaaS/メディア/EC・D2C に「マルチプル法のためユニットエコノミクス系ドライバーはEVに影響しない」注記が表示される(P4-1 追加指示)
- [ ] ドメイン外入力(例: SaaS グロスマージンに120%)→ チャートの代わりにエラー表示
- [ ] SaaS で `マルチプル適用基準=実績(current)` に切替 → `ARR成長率` が「感度なし」グループへ移る

### C-2. 資本政策シミュレーター(C8。全6セクター、VC法セクション直後)

- [ ] 初期保有者行・ラウンド行の追加/編集/削除が動作。**中間行を削除しても後続行の入力値・フォーカスが行ズレしない**(useStableListKeys)
- [ ] 持分推移マトリクス(行=保有者、列=初期+各ラウンド後)が更新される
- [ ] Exit 時の自社実効持分・手取り・期待IRR/MOIC が表示される。自ファンド出資ゼロのとき IRR/MOIC は「—」表示
- [ ] exitEvSource セレクト(悲観/ベース/楽観、既定=ベース)の切替で手取り・IRR/MOIC が変化する

## D. (任意)E2E テストの恒久化

毎回手動で確認する代わりに Playwright テストとして固定する場合:

1. `npm i -D @playwright/test@1.61.1`(バージョンを CLI と揃える。Chromium は導入済みのため再ダウンロード不要)
2. `playwright.config.ts` を追加: `webServer: { command: 'npm run dev', url: 'http://localhost:5173' }`、`use: { browserName: 'chromium' }`。**vitest の対象(`npm run test`)と混ざらないよう** testDir は `e2e/` に分離し、`npm run test:e2e` スクリプトを追加する
3. スペックは §C-1/C-2 の太字項目(行ズレなし・リロード復元・計算不能警告)+ `page.on('console')` でエラーゼロのアサーションを最小セットとする
4. コミットは CLAUDE.md 規約に従い `test(e2e): ...` の単独コミット(エンジン・UIの変更と混ぜない)。package.json 変更を伴うため、実施前にユーザーの了承を得ること
